# 야담 대본 작성 스킬 v0.9

유튜브 채널 "무한야담"의 대본을 **11단계 워크플로우**로 작성한다.
대본 작성 + 프로덕션(TTS/SFX/이미지/비디오) 파이프라인을 통합한 풀스택 버전이다.

## 🔴 최상위 원칙: 궁금증 + 기대감 = 몰입감

> **궁금증과 기대감이 곧 몰입감이다. 몰입감이 이 스킬의 최상위 성공 기준이다.**
>
> "누가 범인인가?", "진실은 무엇인가?"를 시청자가 전(13~17챕터) 중반(15챕터)까지 확신하지 못해야 한다.
> 궁금증이 끊기는 순간 기대감이 사라지고, 몰입이 무너진다.
>
> - 기~승: 의심 대상이 2명 이상 → 시청자가 확신 불가 → 궁금증 유지
> - 전 초반~중반: 거짓 해결 / 반전으로 예측 뒤집기 → 기대감 극대화
> - 전 후반(16~17챕터): 비로소 진실 폭로 → 몰입의 정점
> - **전 중반 이전에 범인/진실이 확정되면 구조적 실패로 간주한다**

## 모드 선택

사용자의 요청에 따라 두 가지 모드 중 하나를 선택한다.

### 🆕 신규 작성 모드 (11단계)

새로운 에피소드를 처음부터 만들 때 사용한다.

```
① 성공요인 분석 → ② 팩트체크 → ③ 추가 정보 검색
→ ④ 시놉시스 작성 → ⑤ 프리플라이트
→ ⑥ 대본(시나리오) 작성 → ⑦ 검토 후 수정
→ ⑧ 스토리보드(CSV) → ⑨ 프로덕션(TTS/SFX/이미지)
→ ⑩ 영상 생성 → ⑪ 업로드 정보
```

### 🔄 리라이팅 모드 (5단계)

기존 대본이 있을 때 — 구조 개선, 궁금증 유지 강화, 서술 품질 향상 등을 위해 다시 쓸 때 사용한다.
"다시 써줘", "리라이팅", "리디자인", "수정해줘", "궁금증이 안 유지돼" 등의 요청 시 이 모드를 사용한다.

```
① 기존 대본 진단 → ② 팩트체크 & 추가 정보 (필요 시)
→ ③ 시놉시스 리디자인 → ④ 프리플라이트
→ ⑤ 대본 리라이팅 → ⑥ 검토 후 수정
```

#### R1단계: 기존 대본 진단

기존 대본 파일(기/승/전/결)을 읽고 다음을 분석한다:

- **궁금증 유지**: 범인/진실이 어느 시점에 드러나는가? 너무 일찍 드러나지 않는가?
- **서스펜스 구조**: 오픈루프, 거짓해결, 패턴파괴 등이 활용되었는가?
- **감정 설계**: Show Don't Tell이 지켜지는가? 패턴 반복은 없는가?
- **구조 균형**: 기승전결 비율, 글자수 배분이 적절한가?
- **캐릭터**: 의심 대상 배치, 반전 캐릭터 설계가 효과적인가?
- **복선**: 설치/회수가 제대로 되었는가?

**출력 형식:**

```
## 기존 대본 진단

### 점수 (각 10점)
- 몰입감: X/10 — ...
- 감동: X/10 — ...
- 재미(궁금증): X/10 — ...

### 핵심 문제점
1. ...
2. ...

### 유지할 점 (잘 된 부분)
1. ...

### 리디자인 방향
- ...
```

**출력 파일**: `R1_진단.md`

#### R2단계: 팩트체크 & 추가 정보 (필요 시)

진단 결과에서 시대고증 오류가 발견되었거나, 리디자인에 새로운 역사적 배경/인물/풍속 정보가 필요할 때 실행한다.

- WebSearch 자동 사용 (별도 허락 불필요)
- 기존 대본의 고증 오류 수정
- 리디자인에 필요한 추가 자료 수집 (원전, 유사 설화, 시대 배경 등)
- 진단에서 문제 없으면 이 단계를 건너뛸 수 있다

**출력 파일**: `R2_팩트체크.md` (필요 시)

#### R3단계: 시놉시스 리디자인

진단 결과를 바탕으로 20챕터 시놉시스를 새로 설계한다.
기존 대본의 좋은 부분은 유지하면서 문제점을 해결하는 방향으로 구조를 잡는다.

핵심 원칙:
- **궁금증이 무너지는 지점**을 찾아서 구조적으로 해결
- 범인/진실 폭로 타이밍을 **전 후반 이후**로 재배치
- 의심 대상 재설계 (미끼 용의자 추가/변경)
- 서스펜스 기법 재배치
- 복선 추적표 재작성

**출력 파일**: `R3_시놉시스_리디자인.md`

#### R4단계: 프리플라이트

5단계와 동일한 체크리스트로 리디자인된 시놉시스를 점검한다.

**출력 파일**: `R4_프리플라이트.md`

#### R5단계: 대본 리라이팅

리디자인된 시놉시스에 따라 기승전결 4파트를 다시 작성한다.
6단계의 모든 규칙(문체, Show Don't Tell, 서스펜스 기법, 글자수 등)을 적용한다.

#### R6단계: 검토 후 수정

7단계와 동일한 기준으로 검토한다. 특히 **진단에서 지적된 문제점이 해결되었는지** 중점 확인한다.

**출력 파일**: `R6_검토.md`

### 1단계: 성공요인 분석

사용자가 레퍼런스(URL, 텍스트, 영상)를 제공하면 분석한다.

#### 분석 항목
- **훅(Hook)**: 첫 30초~1분의 오프닝 구조
- **스토리 구조**: 기승전결 패턴, 긴장-이완 리듬, 반전 타이밍
- **감정 설계**: 어떤 감정을 어느 지점에서 자극하는가
- **궁금증 유지**: 떡밥 배치, 미스터리 구조, 정보 공개 타이밍
- **캐릭터**: 감정이입 포인트, 반전 캐릭터, 의심 대상 배치
- **언어 스타일**: 문장 길이, 종결어미, 반복 어구, 청자 호칭
- **클로징**: 마무리 방식과 다음 영상 유도

#### 출력

```
## 성공 요인 분석

**훅 기법**: ...
**스토리 구조**: ...
**감정 설계**: ...
**궁금증 유지**: ...
**캐릭터 설계**: ...
**언어 스타일**: ...
**클로징 기법**: ...

**→ 핵심 성공 공식 요약**: (1~3줄)
```

레퍼런스가 없으면 주제만 받고 2단계로 넘어간다.

**출력 파일**: `01_분석.md`

### 2단계: 팩트체크

레퍼런스나 주제에 나오는 역사적 사실을 검증한다.

- 조선시대 관직, 신분제도, 생활상 시대 적합성
- 지명, 인물명 실존 여부
- 관습, 풍속, 의례 고증
- WebSearch 자동 사용 (별도 허락 불필요)

#### 출력

```
## 팩트체크 결과

✅ 확인된 사실: ...
⚠️ 수정 필요: (원문) → (정확한 내용)
❓ 불확실 (추가 확인 권장): ...
```

**출력 파일**: `02_팩트체크.md`

### 3단계: 추가 정보 검색

WebSearch로 주제 관련 자료를 수집한다. (자동 실행)

- 원전 자료 (청구야담, 어우야담, 계서야담 등)
- 유사 설화/민담 변이형
- 시대 배경 역사적 사건
- 디테일 자료 (음식, 의복, 건축, 풍속)
- 감초 에피소드

**출력 파일**: `03_자료수집.md`

### 4단계: 시놉시스 작성 ⭐NEW

**`야담_시놉시스_작성_지침.md`를 반드시 읽고 시작한다.**

20챕터 프레임워크에 따라 시놉시스를 작성한다.

#### 20챕터 구조

```
【기】 도입부 (1~5)
├─ 1. 시대적 배경 설정
├─ 2. 주인공 소개
├─ 3. 주변 인물 관계망
├─ 4. 평화로운 일상과 변화의 조짐
└─ 5. 사건 발생 — 일상의 균열

【승】 전개부 (6~12)
├─ 6. 첫 번째 선택과 행동
├─ 7. 이야기 흐름을 바꾸는 존재 등장
├─ 8. 첫 번째 시련과 대응
├─ 9. 첫 번째 반전 — 상황의 변화
├─ 10. 관계의 심화와 변화
├─ 11. 두 번째 시련 — 위기의 심화
└─ 12. 중요한 비밀/정보 발견

【전】 위기부 (13~17)
├─ 13. 주인공의 내적 성장과 변화
├─ 14. 핵심 반전 — 상황 역전
├─ 15. 최대 위기 — 절망의 순간
├─ 16. 결정적 선택과 행동
└─ 17. 절정 — 최종 대결/시험

【결】 결말부 (18~20)
├─ 18. 모든 것의 진실 드러남
├─ 19. 갈등 해소와 새로운 질서
└─ 20. 교훈과 여운
```

#### 시놉시스에서 미리 설계할 것

- **🔴 궁금증 유지 설계 (최우선)**: 시놉시스 단계에서 "시청자가 범인/진실을 확신하는 최초 지점"을 명시하고, 그것이 15챕터(전 중반) 이후인지 반드시 확인
- **감정-행동 매핑**: 주요 감정 장면마다 Show 방법 기획
- **서스펜스 기법 배치**: 어느 챕터에 어떤 기법을 쓸지 설계
  - 기(1~5): 오픈 루프, 정보 격차, 시드 배치
  - 승(6~12): 극적 아이러니, 점진적 해빙, 에스컬레이션
  - 전(13~17): 클리프행어, 타임 프레셔, 거짓 해결, 패턴 파괴
  - 결(18~20): 콜백, 아이러니 역전, 작은 은혜의 복리
- **복선 추적표 초안**: 설치/회수 챕터 미리 설정
- **의심 대상 배치**: 진범 외 최소 2명의 미끼 용의자 설계
- **🔴 반전 갭 설계**: 진범을 시청자가 가장 신뢰하는 인물로 위장 → 폭로 시 갭 극대화. 반전 전까지 신뢰를 쌓는 장면을 충분히 배치
- **시네마틱 포인트**: 슬로모션, 병렬 편집 등 핵심 장면 설계

**출력 파일**: `04_시놉시스.md`

### 5단계: 프리플라이트 체크 ⭐NEW

#### 🔴 절대 금지 확인

| # | 금지 사항 |
|---|----------|
| 1 | 감정 직접 명명 ("슬펐습니다", "기뻤습니다") |
| 2 | 말줄임표 (.., …) |
| 3 | 외래어 (키스, 다크서클 등) |
| 4 | 영어/알파벳 일체 |
| 5 | 아라비아 숫자 (8시 → 여덟 시) |
| 6 | '그', '그녀' 지칭 → 이름/호칭 사용 |
| 7 | "~씨" 호칭 → 낭자, 도령, 마님 등 |
| 8 | "하녀" → 몸종, 침모 |

#### 🟡 시놉시스 구조 점검

- [ ] 의심 대상 최소 2명 배치되었는가?
- [ ] 반전이 2중 이상 설계되었는가?
- [ ] 🔴 궁금증이 전 중반(15챕터)까지 유지되는가? → 시청자가 범인/진실을 확신하는 최초 지점이 15챕터 이후인지 확인
- [ ] 서스펜스 기법이 각 구간에 1개 이상 배치되었는가?
- [ ] 복선 추적표가 완성되었는가?
- [ ] 감정선 곡선이 올라갔다 내려갔다 교차하는가?

#### 복선 추적표 양식

|---|----------|----------|----------|------|

#### 글자수 배분 (적정 범위: 8,000 ~ 12,000자)

#### 문체 규칙

- **시점**: 3인칭 관찰자 시점 일관 유지
- **종결**: "~습니다", "~지요" 존댓말 형식
- **시작**: "옛날 옛적" 또는 "먼 옛날"
- **마무리 CTA**: "다음 영상을 빠르게 만나 보시려면 좋아요와 구독을 눌러 주세요. 그럼 더 재미있는 영상으로 찾아뵙겠습니다. 모두 행복한 하루 보내세요."
- **대사**: 자연스러운 사극 말투 ("~하옵니다", "~이로구나")
- **신분별 화법**: 양반→서민 반말, 서민→양반 극존칭

#### Show, Don't Tell 핵심 규칙

감정을 직접 명명하지 않고 행동/신체반응/환경/대사로 보여준다.

| 감정 | ❌ Tell | ⭕ Show |
|------|--------|--------|
| 슬픔 | "슬펐습니다" | 고개를 숙이다, 입술을 깨물다, 눈시울이 붉어지다 |
| 분노 | "분노했습니다" | 주먹을 불끈 쥐다, 이를 악물다, 목소리가 떨리다 |
| 공포 | "두려웠습니다" | 등줄기에 한기가 흐르다, 다리가 굳다 |
| 사랑 | "사랑했습니다" | 볼이 붉어지다, 눈을 피하다, 심장이 뛰다 |
| 감동 | "감동했습니다" | 눈시울이 뜨거워지다, 목이 메다 |

#### 서스펜스 기법 (대본 적용)

| 기법 | 적용 위치 |
|------|-----------|
| 오픈 루프 | 기~승 초반, 질문을 던지고 답을 2~3챕터 뒤에 |
| 극적 아이러니 | 승, 시청자는 아는데 인물은 모르는 상황 |
| 타임 프레셔 | 전, 시간 제한으로 긴박감 |
| 거짓 해결 | 전, 해결된 것 같다가 더 큰 문제 |
| 콜백 | 결, 초반 대사/장면의 의미 변화 |

#### 도입부(기) — 첫 30초가 승부

- 첫 문장에서 강렬한 상황을 던진다
- "어느 날" 같은 평범한 시작 금지
- 3문장 안에 핵심 갈등이나 미스터리 암시

#### 전개부(승) — 궁금증을 쌓아올린다

- 의심 대상 최소 2명 배치 (미스터리 구조)
- 작은 승리 → 더 큰 위기 패턴 반복
- 진범을 아군/조력자로 위장
- 감정적 유대 형성 장면 (인정, 우정, 효심)

#### 위기/반전(전) — 반전은 2중으로, 갭은 극대로

- 1차 반전: "설마?" + 2차 반전: "이것까지?!"
- **🔴 범인 정체는 전 중반(15챕터) 이후에 폭로** — 그 이전에 확정되면 구조적 실패
- **🔴 반전 갭 원칙**: 시청자가 믿고 있던 것과 진실 사이의 거리(갭)가 클수록 좋다
  - 가장 믿었던 아군이 진범 → 갭 극대 → 충격 극대
  - 가장 의심했던 인물이 사실 보호자 → 이중 반전
  - 반전 전에 그 인물을 시청자가 충분히 신뢰/의심하도록 장면을 쌓아야 갭이 커진다
- 반전 후 감정의 깊이를 주는 장면 (충격 → 배신감 → 분노 → 카타르시스)

#### 결말(결) — 여운을 남긴다

- 복선 회수로 쾌감
- 콜백으로 감동 극대화
- 교훈과 여운을 담는 마지막 문장

#### 매 챕터 전환 시 체크

- [ ] 감정을 직접 명명한 곳이 없는가? (Show, Don't Tell)
- [ ] 시네마틱 기법 1개 이상 활용했는가?
- [ ] 오감 중 2가지 이상 활용했는가?
- [ ] 이전 챕터와 자연스럽게 연결되는가?
- [ ] 글자수 배분을 준수했는가?
- [ ] 복선 추적표가 최신 상태인가?

- [ ] 감정 직접 명명이 없는가? (Show, Don't Tell)

**마무리:**
- [ ] 악인 응징이 통쾌한가?
- [ ] 결말에 여운이 있는가?

### 8단계: 스토리보드 (CSV 생성)

대본을 기반으로 **레퍼런스 CSV**와 **씬 CSV**를 생성한다.
Flow2CapCut MCP의 `get_schema` 도구로 CSV 스키마를 조회하여 정확한 구조를 따른다.

```
flow2capcut MCP: get_schema({ type: "scenes" })      → 씬 CSV 컬럼 확인
flow2capcut MCP: get_schema({ type: "references" })   → 레퍼런스 CSV 컬럼 확인
flow2capcut MCP: get_schema({ type: "prompt-image" }) → 프롬프트 작성 가이드
```

#### 8-1. 레퍼런스 CSV 작성 (`references.csv`)

대본의 등장인물/장소/스타일을 레퍼런스로 정의한다.

| 컬럼 | 설명 |
|------|------|
| `name` | 레퍼런스 이름 (인물명, 장소명) |
| `type` | `character` / `scene` / `style` |
| `prompt` | 영문 이미지 생성 프롬프트 |

**인물(character) 작성 규칙:**
- `solo, single person`으로 시작
- 나이, 성별, 외모, 머리(상투/댕기머리), 의복(저고리+치마/도포 등), 표정 포함
- 조선시대 용어 병기: `topknot (상투)`, `gat hat (갓)`
- 마지막에 `historical Korean costume, no modern clothing`

**장소(scene) 작성 규칙:**
- 시대, 건축양식, 조명, 분위기 포함
- 시간대 변형: `courtyard`, `courtyard_rain`, `courtyard_night`
- 마지막에 `no modern elements`

#### 8-2. 씬 CSV 작성 (`{제목}_scenes.csv`)

| 컬럼 | 필수 | 설명 |
|------|------|------|
| `prompt` | ✅ | 영문 이미지/비디오 프롬프트 |
| `prompt_ko` | | 한글 프롬프트 요약 |
| `subtitle` | | 나레이션/대사 자막 |
| `characters` | | 등장 인물 (쉼표 구분) |
| `scene_tag` | | 장소 태그 (references.csv의 scene name과 매칭) |
| `style_tag` | | 분위기 태그 |
| `shot_type` | | `scene` / `reaction` / `narration` / `dialogue` |
| `duration` | | 씬 길이 (초) |
| `start_time` | | 시작 시간 (초) |
| `end_time` | | 종료 시간 (초) |
| `parent_scene` | | 씬 그룹 ID (S001, S002...) |

#### 씬 분리 규칙
- **한 씬은 15초를 넘기지 않는다** (시청자 집중력 유지)
- 내용적으로 구분되는 단위로 나눈다 (장소, 시간, 행동, 감정 전환 기준)
- 같은 장소라도 감정/행동이 바뀌면 별도 씬으로 분리
- 대사 중심 씬과 묘사 중심 씬을 구분
- 씬당 평균 10초 전후 (20분 영상 기준 약 120~350장면)

#### 8-3. SFX 추출

대본에서 음향효과가 필요한 구간을 추출한다.

| 카테고리 | 예시 |
|----------|------|
| 소품 | 주판 소리, 장부 넘기는 소리 |
| 환경음 | 바람, 빗소리, 새소리, 장터, 풀벌레 |
| 인체 | 숨소리, 한숨, 발소리 |
| 금속/문 | 문 여닫기, 자물쇠 |
| 필기 | 붓으로 쓰는 소리 |
| 군중 | 수군거림, 웅성거림 |

**출력 파일**: `references.csv`, `{제목}_scenes.csv`, `08_sfx_목록.md`

### 9단계: 프로덕션 (TTS / SFX / 이미지)

스토리보드 CSV를 기반으로 오디오와 이미지를 생성한다.

#### 9-1. TTS 대사 음성 생성 (Typecast)

대본에서 인물별 대사를 추출하고 TTS로 생성한다.

```
API: https://api.typecast.ai/v1/text-to-speech
인증: {{HOME}}/.typecast/credentials
```

**대사 데이터 구조:**
```python
# (캐릭터명, 번호, 타임코드, 대사, 감정)
("소은", 1, "01:59", "서른한 푼이요.", "normal")
```

**감정 파라미터:** `normal`, `happy`, `sad`, `angry`

**출력:** `voice_samples/{인물명}/{인물}_{번호}_{타임코드}.mp3`

#### 9-2. SFX 음향효과 생성 (ElevenLabs)

8단계에서 추출한 SFX 목록을 기반으로 음향효과를 생성한다.

```
API: https://api.elevenlabs.io/v1/sound-generation
인증: {{HOME}}/.elevenlabs/credentials
```

**SFX 데이터 구조:**
```python
# (폴더, 파일명, 영문 프롬프트, 길이(초))
("01_주판", "abacus_beads_01.mp3",
 "Wooden abacus beads clicking gently, traditional Korean counting", 3)
```

**출력:** `voice_samples/sfx/{카테고리}/{파일명}.mp3`

#### 9-3. 레퍼런스 이미지 생성 (Flow2CapCut)

```
flow2capcut MCP: load_csv → references.csv 로드
flow2capcut MCP: app_generate_reference → 레퍼런스 이미지 생성
flow2capcut MCP: app_wait_batch → 생성 완료 대기
```

#### 9-4. 씬별 이미지 생성 (Flow2CapCut)

```
flow2capcut MCP: load_csv → 씬 CSV 로드
flow2capcut MCP: app_start_batch → 일괄 생성 시작
flow2capcut MCP: app_wait_batch → 생성 완료 대기
```

생성 후 `list_problem_scenes`로 문제 씬을 확인하고 프롬프트를 수정한다.

**출력:** `footage/` 디렉토리에 이미지/비디오 파일

### 11단계: 유튜브 업로드 정보

## 성공 채널 벤치마크

| 채널 | 대표 영상 | 조회수 | 길이 |
|------|---------|--------|------|
| 설화야담 | 큰 며느리는 땅, 막내 며느리는 시어머니 | 88만 | 1시간40분 |
| 야담쉼터 | 친정 가는 며느리에게 더러운 빨래 뭉치 | 79만 | 1시간52분 |
| 구름야담 | 산에 버려진 관상 보는 천재 아이 | 79만 | 52분 |
| 초화야담 | 5냥에 사온 천재아이 | 64만 | 1시간23분 |
| 조선야담소 | 뚱녀 여종을 20냥에 산 시아버지 | 62만 | 1시간12분 |

### 성공하는 소재 패턴

- **신분 역전**: 거지→부자, 종→상궁, 바보→천재
- **숨겨진 정체**: 버려진 아이가 왕손, 바보 행세한 천재
- **가족 갈등과 화해**: 며느리-시어머니, 형제간 재산 분쟁
- **지혜로운 약자**: 아이, 여성, 하인이 지혜로 위기를 넘김
- **인과응보**: 악행의 대가, 선행의 보상이 극적으로 드러남

### 성공하는 제목 패턴

- 자극적 상황 + 궁금증: "산에 버려진 관상 보는 천재 아이, 조선의 운명을 바꾸다"
- 반전 암시: "모두가 비웃던 8살 꼬마 신랑이 첫날밤 건넨 소름 돋는 한마디"
- 인물 대비: "5냥에 사온 천재아이가 정승의 비극을 막은 사연"

## 전체 진행 원칙

1. 단계 시작 전 현재 단계를 명시한다 (예: `## ▶ 2단계: 팩트체크`)
2. 각 단계 완료 후 "다음 단계(X단계)로 진행할까요?" 라고 물어본다
3. 사용자가 수정 요청을 하면 해당 단계를 재실행한다
4. 웹 검색은 2·3단계에서 자동 사용한다 (별도 허락 불필요)
5. **6단계(대본 작성) 전에 반드시 new_prompt/ 5개 문서를 읽는다**
6. 역사·야담·실화 주제의 경우, 사실 왜곡에 특히 주의한다
7. 🔴 궁금증 유지가 최우선 — 범인/진실이 전 중반(15챕터) 이전에 드러나면 구조적 실패
