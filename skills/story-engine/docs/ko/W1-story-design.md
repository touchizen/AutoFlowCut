# W1: 스토리 설계

이 문서는 story-engine 스킬의 W1(성공요인 분석, 팩트체크, 자료수집) 단계 가이드입니다.

## 환경 사전점검 (시작 전 필수)

W1 작업 시작 전에 파이프라인 전체를 완료할 수 있는 환경인지 확인한다. 여기서 막히면 W1~W4를 낭비한다.

### 필수 항목

| # | 항목 | 확인 방법 | 실패 시 |
|---|------|----------|---------|
| 1 | **AutoFlowCut 앱 실행** | `app_status` MCP 호출 (localhost:3210) | 앱 실행 안내 |
| 2 | **작업 폴더 설정** | `app_list_projects` 응답 확인 | 앱 설정에서 폴더 지정 안내 |
| 3 | **Claude Code MCP 연결** | `mcp__autoflowcut__*` 도구 접근 가능 | MCP 등록/재시작 안내 |

### 선택 항목 (없으면 특정 Wave 불가)

| # | 항목 | 용도 | 확인 경로 | 없을 때 대안 |
|---|------|------|----------|-------------|
| 4 | **ElevenLabs API** | W5 음성/SFX 생성 | `~/.elevenlabs/credentials` | Vrew 수동 / Google AI Studio / W5 스킵 |
| 5 | **Typecast API** | W5 대사 TTS (감정별) | `~/.typecast/credentials` | ElevenLabs 캐릭터 보이스 분리로 대체 |
| 6 | **Google Flow 로그인** | W7 이미지/영상 생성 | AutoFlowCut 앱 내 Flow 탭 로그인 상태 | CapCut에서 수동 이미지 삽입 |
| 7 | **Google AI Studio API** | W5 Gemini TTS (연동 예정) | `~/.google-ai-studio/credentials` | ElevenLabs 또는 Typecast로 대체 |

### 체크 결과 보고

사전점검 결과를 사용자에게 표로 제시하고, **선택 항목 중 누락된 것이 있으면 사용자에게 선택지를 물어본다.**

```
환경 확인 결과:
✅ AutoFlowCut 앱 실행 중
✅ 작업 폴더 설정됨
✅ MCP 연결됨
⚠️ ElevenLabs credentials 없음 → W5 음성/SFX 생성 불가
✅ Google Flow 로그인됨

ElevenLabs 없이 진행 방법:
1. W1~W4까지만 진행, W5는 Vrew로 수동 처리
2. 무료 계정 만들고 credentials 저장 후 계속 (elevenlabs.io 10분 무료)
3. W5를 완전히 스킵하고 CapCut에서 오디오 수동 처리
4. 작업 취소

어떻게 하시겠습니까?
```

**사용자 응답 전에는 W1 본 작업을 시작하지 않는다.**

## [신규] 경로

### 1. 팩트체크

레퍼런스나 주제에 나오는 역사적 사실을 검증한다.

- 조선시대 관직, 신분제도, 생활상 시대 적합성
- 지명, 인물명 실존 여부
- 관습, 풍속, 의례 고증
- WebSearch 자동 사용 (별도 허락 불필요)

#### 출력

```
## 팩트체크 결과

✅ 확인된 사실: ...
⚠️ 수정 필요: (원문) → (정확한 내용)
❓ 불확실 (추가 확인 권장): ...
```

**출력 파일**: `02_팩트체크.md`

### 2. 추가 정보 검색

WebSearch로 주제 관련 자료를 수집한다. (자동 실행)

- 원전 자료 (청구야담, 어우야담, 계서야담 등)
- 유사 설화/민담 변이형
- 시대 배경 역사적 사건
- 디테일 자료 (음식, 의복, 건축, 풍속)
- 감초 에피소드

**출력 파일**: `03_자료수집.md`

## [리라이팅] 경로

### 성공요인 분석

사용자가 레퍼런스(URL, 텍스트, 영상)를 제공하면 분석한다.

#### 분석 항목
- **훅(Hook)**: 첫 30초~1분의 오프닝 구조
- **스토리 구조**: 기승전결 패턴, 긴장-이완 리듬, 반전 타이밍
- **감정 설계**: 어떤 감정을 어느 지점에서 자극하는가
- **궁금증 유지**: 떡밥 배치, 미스터리 구조, 정보 공개 타이밍
- **캐릭터**: 감정이입 포인트, 반전 캐릭터, 의심 대상 배치
- **언어 스타일**: 문장 길이, 종결어미, 반복 어구, 청자 호칭
- **클로징**: 마무리 방식과 다음 영상 유도

#### 출력

```
## 성공 요인 분석

**훅 기법**: ...
**스토리 구조**: ...
**감정 설계**: ...
**궁금증 유지**: ...
**캐릭터 설계**: ...
**언어 스타일**: ...
**클로징 기법**: ...

**→ 핵심 성공 공식 요약**: (1~3줄)
```

**출력 파일**: `01_분석.md`

## 성공 채널 벤치마크

| 채널 | 대표 영상 | 조회수 | 길이 |
|------|---------|--------|------|
| 설화야담 | 큰 며느리는 땅, 막내 며느리는 시어머니 | 88만 | 1시간40분 |
| 야담쉼터 | 친정 가는 며느리에게 더러운 빨래 뭉치 | 79만 | 1시간52분 |
| 구름야담 | 산에 버려진 관상 보는 천재 아이 | 79만 | 52분 |
| 초화야담 | 5냥에 사온 천재아이 | 64만 | 1시간23분 |
| 조선야담소 | 뚱녀 여종을 20냥에 산 시아버지 | 62만 | 1시간12분 |

### 성공하는 소재 패턴

- **신분 역전**: 거지→부자, 종→상궁, 바보→천재
- **숨겨진 정체**: 버려진 아이가 왕손, 바보 행세한 천재
- **가족 갈등과 화해**: 며느리-시어머니, 형제간 재산 분쟁
- **지혜로운 약자**: 아이, 여성, 하인이 지혜로 위기를 넘김
- **인과응보**: 악행의 대가, 선행의 보상이 극적으로 드러남

### 성공하는 제목 패턴

- 자극적 상황 + 궁금증: "산에 버려진 관상 보는 천재 아이, 조선의 운명을 바꾸다"
- 반전 암시: "모두가 비웃던 8살 꼬마 신랑이 첫날밤 건넨 소름 돋는 한마디"
- 인물 대비: "5냥에 사온 천재아이가 정승의 비극을 막은 사연"
