## ★ 대본 primary review lens — 시청자 retention

> SKILL.md 핵심 원칙: **궁금증 + 기대감 = 몰입도. 모든 wave의 최상위 평가 기준.**

W3 대본 검토는 "재미있는가"가 아니라 **"시청자가 다음 단락으로 넘어가게 만드는가"**로 한다. (자세한 review dimension 참조: yadam은 `meta-prompts/yadam/yadam-narrative-guide.md`, dark-history는 `meta-prompts/dark-history/narrative_techniques.md`, bespoke는 `meta-prompts/bespoke/{lang}/narrative_techniques.md` — 모두 § Review dimensions / 리뷰 차원 참조. E0~E3 (engagement)가 primary, C1~C10 (craft)는 subordinate.)

Engagement primary 4 항목 (E0~E3):
- **E0. 궁금증 유지** — 매 단락이 시청자에게 "다음에 뭐?" / "이건 뭐?" 질문을 던지는가. 3 단락 연속 질문 없으면 drop-off zone.
- **E1. 기대감 누적** — 매 단락이 구체적 예측을 심거나 키우는가. 분위기만 깔고 payoff 없으면 wasted space.
- **E2. 몰입도 곡선 일치** — 단락 단위 페이싱이 시놉시스 챕터 곡선과 맞는가.
- **E3. Drop-off zone 없음** — 시청자가 정신적으로 떠날 만한 beat 명시 식별 → 재작성 (편집 trim 아님).

Craft 점수가 9.5 넘어도 E0~E3 한 항목이라도 fail이면 revise. Craft가 engagement 부족을 보상하지 않는다.

## 대본(시나리오) 작성

**선택된 장르의 대본 작성 가이드라인 3개를 반드시 읽고 시작한다** (위 "참조 문서" 표 참조):
- **yadam**: `meta-prompts/yadam/yadam-scenario-guide.md`, `yadam-narrative-guide.md`, `yadam-suspense-techniques.md`
- **dark-history**: `meta-prompts/dark-history/screenplay_guidelines.md`, `narrative_techniques.md`, `suspense_techniques.md`
- **bespoke (맞춤형)**: `meta-prompts/bespoke/{lang}/screenplay_guidelines.md`, `narrative_techniques.md`, `suspense_techniques.md` + `_story_source/_meta_supplement.md`

`{lang}`은 STATE.md의 "Output language:" 필드 (`ko` 또는 `en`)에서 해석.

시놉시스를 바탕으로 **4파트(기/승/전/결 또는 setup/rising/crisis/resolution)를 먼저 작성**한 다음, **전체 흐름을 본 상태에서 Hook을 마지막에 작성**하고, 그 후 통합 검토한다.

### 작성 순서

```
기 → 승 → 전 → 결 → 훅(마지막) → (모두 완성 후) 검토
```

**Hook을 마지막에 쓰는 이유** — 콜드 오픈의 유일한 목적은 첫 20–30초 안에
시청자의 궁금증/기대감을 즉시 점화하는 것. 어떤 hook 유형(Act III에서
가져온 flash-forward / mystery opening / provocative statement / sensory
immersion)이 가장 강력한지는 전체 arc를 다 본 다음에야 판단 가능. 훅을 먼저
쓰면 "옛날 옛적에…" 같은 평범한 도입부가 되기 쉬움.

**Hook은 별도 파일** — `{title}_hook.md`. `{title}_기.md` (yadam) /
`{title}_part1_setup.md` (dark-history / bespoke) 안에 hook 내용을 중복으로
적지 않는다 — part1은 hook 이후의 본격적인 내러티브로 시작. Hook을 분리하면 작가는 hook만 단독으로 반복 다듬기 가능
(다른 part 재생성 없이 hook A/B 테스트), 리뷰어는 영상 전체 retention의 핵심
변수인 첫 30초에 집중 가능.

**Hook 길이 타깃**: 약 20초 분량의 나레이션 (80–130자 한국어 / 50–80 단어 영어).

### W3 산출물

W3 단계는 **스크립트 5개 + 리뷰 1개** 만 산출한다. 그 외는 모두 다른 wave 책임.

```
ep{번호}/                    ← yadam 기준 (다른 장르는 위 매핑 표 참조)
├── {제목}_기.md           ← 도입부 (hook 이후)
├── {제목}_승.md           ← 전개부
├── {제목}_전.md           ← 위기/반전
├── {제목}_결.md           ← 결말
├── {제목}_hook.md         ← 콜드 오픈 (약 20초, 마지막에 작성, 모든 장르 ASCII 공통)
└── 07_검토.md             ← 통합 검토
```

W1/W2 산출물(`01_분석.md` ~ `05_프리플라이트.md`)은 W3에서 **읽기 전용**으로 참조한다. 다운스트림 산출물(`segments_{파트}/`, `final_{파트}.mp3`, `{제목}_scenes.csv`, `media/final_full.mp3` 등)은 W4–W8에서 생성되며 **W3 책임이 아니다**. 전체 wave별 산출물 매핑은 `workflows/execute-pipeline.md` § Wave I/O contract (단일 권위 출처) 참조.

### 글자수 배분 (적정 범위: 6,000 ~ 12,000자, 영상 길이 약 18~36분)

| 구간 | 글자 수 (9,000자 기준) | 비율 |
|------|------------------------|------|
| 【기】 도입부 | 1,350자 | 15% |
| 【승】 전개부 | 2,700자 | 30% |
| 【전】 위기부 | 3,150자 | 35% |
| 【결】 결말부 | 1,800자 | 20% |
| **합계** | **9,000자** (중간값) | 100% |

> **유의**: 6,000자 미만이면 몰입도가 얕아지고, 12,000자 초과면 집중 유지가 어렵다. **단, 몰입도 유지가 되는 경우에 한해 이 범위를 벗어나도 허용한다.**
>
> 글자수 확인: `LC_ALL=ko_KR.UTF-8 wc -m`

### 문체 규칙

- **시점**: 3인칭 관찰자 시점 일관 유지
- **종결**: "~습니다", "~지요" 존댓말 형식
- **본편(`_기.md`) 시작**: "옛날 옛적" 또는 "먼 옛날" (야담 전통 화법). ⚠ Hook 파일(`{title}_hook.md`)에는 사용 금지 — Hook은 [`_common/hook_principles.md`](../../meta-prompts/_common/hook_principles.md) 규칙 적용 (generic opening 회피).
- **마무리 CTA**: "다음 영상을 빠르게 만나 보시려면 좋아요와 구독을 눌러 주세요. 그럼 더 재미있는 영상으로 찾아뵙겠습니다. 모두 행복한 하루 보내세요."
- **대사**: 자연스러운 사극 말투 ("~하옵니다", "~이로구나")
- **신분별 화법**: 양반→서민 반말, 서민→양반 극존칭

### Show, Don't Tell 핵심 규칙

감정을 직접 명명하지 않고 행동/신체반응/환경/대사로 보여준다.

| 감정 | Tell | Show |
|------|------|------|
| 슬픔 | "슬펐습니다" | 고개를 숙이다, 입술을 깨물다, 눈시울이 붉어지다 |
| 분노 | "분노했습니다" | 주먹을 불끈 쥐다, 이를 악물다, 목소리가 떨리다 |
| 공포 | "두려웠습니다" | 등줄기에 한기가 흐르다, 다리가 굳다 |
| 사랑 | "사랑했습니다" | 볼이 붉어지다, 눈을 피하다, 심장이 뛰다 |
| 감동 | "감동했습니다" | 눈시울이 뜨거워지다, 목이 메다 |

### 서스펜스 기법 (대본 적용)

| 기법 | 적용 위치 |
|------|-----------|
| 오픈 루프 | 기~승 초반, 질문을 던지고 답을 2~3챕터 뒤에 |
| 극적 아이러니 | 승, 시청자는 아는데 인물은 모르는 상황 |
| 타임 프레셔 | 전, 시간 제한으로 긴박감 |
| 거짓 해결 | 전, 해결된 것 같다가 더 큰 문제 |
| 콜백 | 결, 초반 대사/장면의 의미 변화 |

### 정보 공개 원칙 — 조금씩 풀기

나레이터가 친절하게 해설하면 시청자의 추론 기회를 빼앗는다. **정보는 일부러 덜 알려준다.**

| 친절한 해설 (나쁜 예) | 조금씩 풀기 (좋은 예) |
|------------------------|----------------------|
| "최씨가 조작한 것이 아니었습니다. 누군가의 조작을 감지하고 혼자서 기록해두고 있었다는 뜻이었지요." | "최씨 장부에는 원래 금액이 적혀 있었지요. 소은의 손이 멈추었습니다." |
| "소은은 그때 아무 이상함도 느끼지 못했습니다." (의도 해설) | 질문 대사만 남기고, 시청자가 눈치채게 두기 |
| "이서방이 전부가 아니었습니다." (결론을 대사로 말함) | 차액이 다시 나기 시작 → 소은의 입술이 굳는 행동만 |
| 격언/교훈을 여러 번 반복 ("실수는 고르지 않고, 의도는 고르다") | 격언은 설치 1회 + 회수 1회만. 중간 반복 금지 |

**핵심**: 나레이터는 사실만 전달하고, **의미/해석은 시청자가 추론**하게 한다. "뭔가 이상했다" → 나중에 공개. 바로 설명하지 않는다.

### 해결/액션 장면 원칙 — 빠르고 강하게

위기 해결, 구출, 대결 장면은 **서술 설명을 줄이고 액션 중심**으로 쓴다.

| 서술형 (느린 예) | 액션형 (빠른 예) |
|-------------------|-------------------|
| "최씨에게 곳간 열쇠를 구했으나, 곽 주사가 미리 빼돌린 뒤였습니다." | "열쇠가 없었습니다. 복돌이가 쇠지렛대를 끌고 왔지요. 빗장이 비틀어졌습니다." |
| "소은이 어둠 속에서 장부를 다시 한번 자세히 살펴보았습니다." | "소은이 장부를 펼쳤습니다. 주판을 무릎에 올렸지요. 달그락, 달그락." |
| 화해/고백 대사가 3~4문장 | 핵심 대사 1~2문장 + 행동/신체반응 |

**핵심**: 위기→해결 구간은 **짧은 문장 + 동사 중심 + 소리/움직임**으로 템포를 올린다. 설명이 들어가면 긴장이 풀린다.

### 도입부(기) — 첫 30초가 승부

- 첫 문장에서 강렬한 상황을 던진다
- "어느 날" 같은 평범한 시작 금지
- 3문장 안에 핵심 갈등이나 미스터리 암시

#### Hook 오프닝 갤러리 (작동하는 첫 3문장)

**예시 1 — 결과 선공개 + 원인 궁금증**
> 그날 저녁, 연안 이씨 종가의 곳간에서는 쌀 다섯 섬이 사라졌지요.
> 자물쇠는 멀쩡했고, 문살에도 흠집 하나 없었습니다.
> 그런데 사라진 쌀을 찾아낸 건, 글도 모르는 여덟 살 아이였습니다.

**작동 원리**: 결과(쌀 도난 해결)를 먼저 던지고, 해결자의 정체(8살 문맹 아이)가 반전적. "어떻게 글도 모르는 아이가?"라는 질문이 자동 생성.

---

**예시 2 — 인물의 마지막 말 + 수수께끼**
> 소은이 세상을 떠나기 직전 남긴 말은 단 한 마디였습니다.
> "그 장부는 불태워 달라고 했소."
> 그러나 마을 사람 누구도, 소은의 장부가 어디 있는지 알지 못했지요.

**작동 원리**: 마지막 말 + 그 말의 의미 모름. 오픈 루프 강력. "장부에 뭐가 있었길래?"

---

**예시 3 — 이례적 행동 포착**
> 그해 가을, 양반 김 대감은 하루아침에 자신의 노비 문서를 모두 불태웠습니다.
> 그러나 그 불길이 사라지기도 전, 자기 자식에게 한마디를 남기고 방에 들어갔지요.
> 그 방에서 김 대감이 다시 나온 것은, 꼬박 이레가 지난 뒤였습니다.

**작동 원리**: 이례적 행동(노비 해방) + 그 이후 더 이례적 행동(칠일 침묵). 두 겹의 궁금증.

---

**예시 4 — 시점 이동 + 대조**
> 소년은 말 없이 밥 한 그릇을 받아먹었습니다.
> 서른 해가 지난 뒤, 그 소년은 임금 앞에 서 있었지요.
> 임금은 그날의 밥 한 그릇이, 이 나라 운명을 바꾼 밥이었다는 것을 아직 알지 못했습니다.

**작동 원리**: 평범한 장면 → 30년 점프 → 극적 결과 암시. "어떻게 밥 한 그릇이?"

---

**예시 5 — 부정형 + 수수께끼**
> 그날 밤 삼경, 동네 개들이 단 한 마리도 짖지 않았습니다.
> 수백 년을 그 마을에 살아온 노인들도, 그런 고요는 처음이었지요.
> 다음 날 아침, 이방의 집 앞에는 검은 천으로 싸인 상자가 놓여 있었습니다.

**작동 원리**: "없다"(개들이 짖지 않음) = 이례성. 불길한 정적 + 아침의 상자. 어떤 공포가 오고 있는가?

---

#### 실패하는 Hook 예시 (고치기 전후)

**실패 1 — 평범한 시작**
> ❌ 옛날 옛적, 한 마을에 노부부가 살고 있었습니다. 노부부는 가난했지만 서로 아끼며 살았지요. 어느 날, 마당에 이상한 일이 벌어졌습니다.

**문제**: "옛날 옛적", "어느 날" — 시청자가 즉시 이탈. 첫 3문장에 훅이 없음.

> ✅ 노부부의 마당에서 밤마다 흙이 사라지고 있었습니다. 처음엔 한 줌, 그다음엔 한 삽, 이윽고 한 바지게씩. 그러나 새벽이 되면, 마당은 언제나 원래 자리에 있었지요.

**수정**: 평범한 시작 제거 + 구체적 이상현상(흙 사라짐) + 이례성(복구됨) + 규모 확대(한 줌→바지게).

---

**실패 2 — 정보 과다**
> ❌ 조선 숙종 27년, 한양 북촌에 최씨라는 서른다섯 살 과부가 살았습니다. 남편을 일찍 여의고 장사로 먹고살았는데, 성품이 깐깐하고 계산이 밝았지요. 그녀는 장부를 매일 쓰는 습관이 있었는데, 그 장부가 어느 날 사라졌습니다.

**문제**: 배경 설명 3문장 → 시청자는 이미 이탈. 첫 문장부터 미스터리가 없음.

> ✅ 최씨의 장부가 사라졌을 때, 그녀는 울지 않았습니다. 대신 대문을 걸어 잠그고, 사흘간 아무도 들이지 않았지요. 그 사흘이 지나, 최씨가 대문을 열었을 때 — 손에 들린 것은 똑같이 생긴 새 장부였습니다.

**수정**: 인물 반응(울지 않음, 문 잠금) → 궁금증. 사흘 후 나온 것(새 장부) → 더 큰 미스터리.

#### 서스펜스 기법별 Before / After

**1. 오픈 루프 (질문 던지고 답을 늦춤)**

> ❌ **닫힌 루프 (나쁜 예)**
> "소은은 장부를 펼쳤습니다. 그리고 바로 이서방이 가져간 것을 알아챘지요."

> ✅ **오픈 루프 (좋은 예)**
> "소은은 장부를 펼쳤습니다. 세 번을 확인했지요. 네 번째 주판알이 다시 돌아갈 때, 그녀의 입술이 굳었습니다."

**차이**: 나쁜 예는 즉시 답을 줌. 좋은 예는 행동만 보여주고 답은 나중에.

---

**2. 극적 아이러니 (시청자는 알고 인물은 모름)**

> ❌ **긴장 없음**
> "곽 주사는 최씨의 장부를 조작했습니다. 최씨는 그 사실을 모른 채 곽 주사에게 곡식을 더 내주었지요."

> ✅ **극적 아이러니**
> "최씨는 곽 주사가 건네는 장부를 받아들었습니다. 숫자는 맞았지요. 다만, 시청자는 이미 알고 있었습니다 — 그 장부가 어젯밤 불빛 아래에서 고쳐졌다는 것을."

**차이**: 나쁜 예는 나레이터가 정리. 좋은 예는 시청자에게만 정보 공개, 인물의 평온한 행동이 긴장을 낳음.

---

**3. 타임 프레셔 (시간 제한)**

> ❌ **시간 감각 없음**
> "소은은 장부의 비밀을 풀어야 했습니다. 그러나 쉽지 않은 일이었지요."

> ✅ **타임 프레셔**
> "닭이 세 번 울기 전에 — 소은은 이 장부의 숨은 계산을 찾아내야 했습니다. 첫 홰가 멀리서 울었지요. 그녀의 손이 빨라졌습니다."

**차이**: 좋은 예는 구체적 데드라인(닭 세 번) + 이미 시작된 카운트다운(첫 홰).

---

**4. 거짓 해결 (해결된 것 같은데 아님)**

> ❌ **단선적**
> "범인은 이서방으로 밝혀졌습니다. 모두가 안도했지요. 그러나 진짜 범인은 따로 있었습니다."

> ✅ **거짓 해결**
> "이서방이 끌려 나갔을 때, 마을은 잠시 숨을 돌렸습니다. 곽 주사도 고개를 끄덕였지요. 바로 그때 — 소은의 문밖에, 새 장부 한 권이 놓여 있었습니다."

**차이**: 좋은 예는 해결의 '공기'를 보여주고(안도), 다음 장의 씨앗만 투하.

---

**5. 콜백 (초반 요소의 의미 변화)**

> ❌ **연결 없음**
> "이야기는 끝났습니다. 소은은 평온한 삶을 되찾았지요."

> ✅ **콜백**
> "서른 해가 흘렀습니다. 마을에 새 주판 장인이 들어왔지요. 그의 주판 소리를 듣고 — 소은의 손자는 이상한 기시감을 느꼈습니다. 어디선가 들어본 것 같은, 네 번의 달그락."

**차이**: 초반의 "네 번째 주판알"이 30년 후 손자에게 돌아옴. 의미가 증폭.

#### 몰입 붕괴 신호 (검토 시 체크)

다음 신호가 나타나면 해당 구간은 즉시 수정한다.

| 신호 | 증상 | 수정 방향 |
|------|------|----------|
| 설명 문단 | "그것은 ~때문이었지요" 패턴이 연속 | 행동/반응만 남기고 설명 삭제 |
| 감정 명명 | "슬펐습니다", "기뻤습니다" | Show로 변환 |
| 정보 한꺼번에 | 한 챕터에 3개 이상의 비밀 공개 | 2~3챕터로 분산 |
| 대사 과다 | 5문장 이상 대화 지속 | 중간에 행동/묘사 삽입 |
| 반복 패턴 | "마을 사람은 몰랐습니다"가 3회 이상 | 다른 표현으로 교체 |
| 평탄한 감정선 | 긴장-이완이 없이 같은 톤 | 각 챕터에 감정 전환점 1개 |

### 전개부(승) — 궁금증을 쌓아올린다

- 의심 대상 최소 2명 배치 (미스터리 구조)
- 작은 승리 → 더 큰 위기 패턴 반복
- 진범을 아군/조력자로 위장
- 감정적 유대 형성 장면 (인정, 우정, 효심)

### 위기/반전(전) — 반전은 2중으로, 갭은 극대로

- 1차 반전: "설마?" + 2차 반전: "이것까지?!"
- **범인 정체는 전 중반(15챕터) 이후에 폭로** — 그 이전에 확정되면 구조적 실패
- **반전 갭 원칙**: 시청자가 믿고 있던 것과 진실 사이의 거리(갭)가 클수록 좋다
  - 가장 믿었던 아군이 진범 → 갭 극대 → 충격 극대
  - 가장 의심했던 인물이 사실 보호자 → 이중 반전
  - 반전 전에 그 인물을 시청자가 충분히 신뢰/의심하도록 장면을 쌓아야 갭이 커진다
- 반전 후 감정의 깊이를 주는 장면 (충격 → 배신감 → 분노 → 카타르시스)

### 결말(결) — 여운을 남긴다

- 복선 회수로 쾌감
- 콜백으로 감동 극대화
- 교훈과 여운을 담는 마지막 문장

### 매 챕터 전환 시 체크

- [ ] 감정을 직접 명명한 곳이 없는가? (Show, Don't Tell)
- [ ] 시네마틱 기법 1개 이상 활용했는가?
- [ ] 오감 중 2가지 이상 활용했는가?
- [ ] 이전 챕터와 자연스럽게 연결되는가?
- [ ] 글자수 배분을 준수했는가?
- [ ] 복선 추적표가 최신 상태인가?

## 검토 후 수정 (반복)

작성된 대본을 자체 검토 → subagent 재검토 → 수정을 **수정사항이 없을 때까지 반복**한다. (최대 5라운드, 목표 평균 9.5점)

```
┌─ 1. 자체 검토 + 수정사항 목록 작성
│     ▼
│  수정사항 있음? → YES → 대본 수정
│                         ▼
│                  subagent 재검토 (대본 파일 직접 읽기)
│                         ▼
│                  수정사항 있음? → YES → 수정 → 재검토 (반복)
│                                → NO  → 루프 종료
│                → NO  → 루프 종료
│
│  ※ 최대 5라운드. 초과 시 사용자에게 보고
│
└─ 외부 검토 (선택사항): 사용자 요청 시에만 Gemini/Codex 실행
```

### 평가 기준 (각 10점, 목표 평균 9.5)

> **궁금증 + 기대감 = 몰입감** — 몰입감이 최상위 기준이다.
> 궁금증이 끊기면 기대감이 사라지고, 몰입이 무너진다.

| 항목 | 체크 포인트 | 우선순위 |
|------|-----------|----------|
| **몰입감** | 첫 30초 훅, 궁금증 지속, 지루한 구간 없음, 반전 충격도 | 최우선 |
| **감동** | 감정이입 깊이, 눈물/분노/카타르시스 순간, 여운 | 높음 |
| **재미** | 대사 매력, 캐릭터 개성, 전개 속도, 예측불가능성 | 높음 |

**마무리:**

- [ ] 결말에 여운이 있는가?

### 외부 AI 검증 (선택사항)

> **선택사항**: 사용자가 명시적으로 요청할 때만 실행한다. 시간이 오래 걸리므로 기본은 자체 검토만 수행.

사용자가 외부 검증을 요청하면, Gemini-cli와 Codex MCP에 Agent를 병렬로 띄워서 검증을 요청한다.

**Codex MCP:**
```
cwd: "{작업폴더}/ep{번호}_{slug}/_story_source"
```
- 텍스트를 인라인으로 붙여넣지 말고, 반드시 cwd로 디렉토리 경로를 전달
- 평가 요청: 몰입감/감동/재미 각 10점 + 궁금증 유지 구간 분석

**Gemini-cli:**
```
mcp__gemini-cli__ask-gemini
```
- 대본 파일 경로를 전달하고 피드백 요청

- 타당한 지적 → 수정 반영

**출력 파일**: `07_검토.md` (자체 검토 + 외부 피드백 통합)
